Example Whirlycache Config File

Whirlycache is configurable using the whirlycache.xml file. This file should reside somewhere in your CLASSPATH. If it cannot be found, the whirlycache-default.xml file is loaded from the whirlycache jar file with some reasonably intelligent defaults.

You should, however, create your own whirlycache.xml file by simply copying the sample below and modifying it as necessary. 

<whirlycache>

	<default-cache>default</default-cache>

	<cache name="default">
		<backend>com.whirlycott.cache.impl.ConcurrentHashMapImpl</backend>
		<tuner-sleeptime>10</tuner-sleeptime>
		
		<!-- evicts oldest items when pruning -->
		<!-- <policy>com.whirlycott.cache.policy.FIFOMaintenancePolicy</policy> -->
		<!-- evicts least recently used items when pruning -->
		<!-- <policy>com.whirlycott.cache.policy.LRUMaintenancePolicy</policy> -->
		<!-- evicts least frequently used items when pruning -->
		<policy>com.whirlycott.cache.policy.LFUMaintenancePolicy</policy>
		
		<maxsize>100000</maxsize>
	</cache>

	<cache name="secondone">
		<backend>com.whirlycott.cache.impl.ConcurrentHashMapImpl</backend>	
		<tuner-sleeptime>10</tuner-sleeptime>
		<policy>com.whirlycott.cache.policy.FIFOMaintenancePolicy</policy>
		<maxsize>100</maxsize>
	</cache>
	
	<cache name="third">
		<backend>com.whirlycott.cache.impl.ConcurrentHashMapImpl</backend>
		<tuner-sleeptime>10</tuner-sleeptime>
		<policy>com.whirlycott.cache.policy.FIFOMaintenancePolicy</policy>
		<maxsize>3000</maxsize>
	</cache>
	
</whirlycache>
